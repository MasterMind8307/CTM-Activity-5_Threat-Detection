# CTM-Activity-5_Threat-Detection
Threat Detection

**Objective**
Analyze captured network traffic logs and identify potential security incidents involving:

Brute Force Attack
Phishing Activity
Man-in-the-Middle (MITM) Attack
Port Scanning
Malware Deployment

**Trainees must:**

Review the logs
Identify suspicious events
Gather evidence
Determine impact and severity
Create an incident report
Recommend escalation actions


**Instructions for Trainees**

**Phase 1: Log Review**
Review all 100 network logs provided.
Identify:

Unusual authentication attempts
Connections to suspicious domains/IPs
ARP spoofing indicators
Sequential port probes
Malware download or command-and-control communication


**Phase 2: Threat Classification**
For each suspicious event:

**CategoryExample:** 
IndicatorsBrute ForceMultiple failed logins
Phishing Suspicious email links
MITMDuplicate ARP replies
Port ScanSequential ports scanned
MalwareMalicious executable download

**Phase 3: Evidence Collection
Collect the following:**

Packet Number
Timestamp
Source IP
Destination IP
Port
Protocol/Service
Event Description


**Phase 4: Incident Analysis**
Document:

Guide Question:
What Happened?
Describe the observed activity.

Evidence
Reference packet numbers and timestamps.

Impact
Describe potential impact to:
Confidentiality
Integrity
Availability

Severity
Use:
Low
Medium
High
Critical


**Phase 5: Recommendations**
Recommend actions such as:

Block IP
Reset credentials
Isolate host
Conduct malware scan
Escalate to Incident Response Team

**Expected Outcome:**
Trainees should identify at least 10 incidents, produce evidence-backed analysis, classify severity, and recommend escalation to the Incident Response Team for High and Critical events.
